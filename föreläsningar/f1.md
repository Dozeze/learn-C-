## Basics

C++ Läser kod uppifrån ner. Ifall du försöker kalla något som existerar längre ner i koden så kommer du att få ett fel.

## Funktioner som är beroende av varandra

Låt oss anta att vi har en funktion $f$ som kallar en funktion $g$. Men funktionen $f$ är högre upp än $g$ i koden. Då kommer det att bli fel. ExempelL

```cpp
//...
int f(int x) {
  if (x > 0) 
    return 5;
  else
    return g();
}

int g() {
  return f(1);
}
```

För att detta ska fungera så vill man antingen deklarera att funktionen $g$ existerar i början av koden. Det vill säga - lägga till:

```cpp
int g();
```

längst upp (vid //...). Alternativt (bäst) så skapar man en header - dvs en ".h"-fil med alla funktioner man vill deklarera. Det vill säga - int g(); ,... etc.


## Klasser och strukturer (class, struct)

Klasser och struct är väldigt lika varandra. Den används för ... . Den största skillnaden mellan klasser och strukturer är att ...

Här är ett exempel på en klass:


```cpp
int g();
```

Här är ett exempel på en struktur:


```cpp
int g();
```

Notera att ifall vi skapar ett program ... Skillnaden blir då:


```cpp
int g();
```

Alltså - ...
